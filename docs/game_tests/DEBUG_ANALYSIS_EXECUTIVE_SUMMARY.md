## Root Cause Classification

### 🖼️ Texture Format Support (3 games)
**Games:** TWA (11002), Vortex (12345), Bejeweled (55555)

| Aspect | Working Games | Failing Games |
|--------|--------------|---------------|
| Format | RGB565 (0x1907) | A8 (0x1906) |
| Example | Cubis 2: 320x240 RGB565 | TWA: 2x2 A8 |
| Status | ✅ Handled | ❌ Unsupported |

**Evidence:**
```
# Cubis 2 (WORKS)
live_upload format=Some(Rgb565) src_fmt=0x1907

# TWA (FAILS)  
live_upload format=Some(A8) src_fmt=0x1906
draw1 skipped: no live upload matched UV span Some((149, 75))
```

**Fix Required:** Add A8 pixel format support in `live_gl.rs`

---

### 🎰 Frame Loop Strictness (3 games)
**Games:** Sudoku (50513), Solitaire (50514), Zuma (44444)

**Issue:** Frame diagnostic anomaly detection too strict

**Evidence:**
```
frame_diag anomalies_so_far=2 latest=unexpected_draw_count
ordinal=157 draws=1 (steady=4)
```

**Problem:** Games intentionally send 1 draw per frame, but diagnostic expects steady=4

**Fix Options:**
1. Add game-specific draw profiles
2. Make anomaly detection configurable via env var
3. Relax steady-state requirements

---

### 💀 CPU Crash - Divide by Zero (1 game)
**Game:** Lost (1B200)

**Crash Sequence:**
1. Loads `rserver.bin` (105KB) successfully
2. Initializes audio subsystem
3. **CRASH:** Divide by zero exception

**Evidence:**
```
svc: putchar '\n'
svc: putchar 'Arithmetic exception: Divide By Zero'
```

**Root Cause:** Uninitialized variable in audio/timing code

**Fix Required:** 
- Identify and patch the division
- Or add ARM exception handling to emulator

---

### 📥 Empty Save File (2 games)
**Games:** Sudoku, Solitaire

**Issue:** Loading 0-byte `.clicky-saves/savefile.dat`

```
AsyncFileIO:3 loaded .clicky-saves/savefile.dat (0 bytes, requested 17228)
```

May contribute to initialization failures.
